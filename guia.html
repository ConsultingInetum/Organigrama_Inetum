Guía de mantenimiento del organigrama

Todo lo que necesitas para actualizar el organigrama público cuando alguien entra, sale o cambia en el equipo. Pensado para hacerse en menos de 10 minutos, sin conocimientos técnicos.

01 · INTRODUCCIÓN

Cómo funciona el organigrama

El organigrama es una web estática que muestra los datos de un archivo data.json. Ese JSON se genera a partir de la Microsoft List + carpeta de fotos. Cuando algo cambia en la Lista, regeneras el JSON y lo subes a GitHub. No hay servidor, no hay base de datos, no hay login.

Los componentes en detalle

🗂️Microsoft List

La fuente de verdad. Lista en SharePoint donde se mantienen todas las personas, roles, equipos, ubicaciones, etc. Solo tú y el equipo TWWW pueden editar.

📷Carpeta de fotos

Carpeta local con las fotos de cada persona. El nombre del archivo debe coincidir con el ID M365 de la persona en la Lista.

⚙️conversor.html

Una página HTML que procesa el CSV exportado + las fotos y genera el data.json. Funciona 100% en tu navegador, sin enviar nada a ningún servidor.

📦data.json

El archivo que contiene todos los datos del organigrama (incluidas las fotos en base64). Es lo único que cambia cuando actualizas. Pesa ~1-3 MB con fotos.

🐙GitHub Pages

Hosting gratuito que sirve el organigrama como una URL pública. Cualquiera con el link puede ver el organigrama, pero solo tú puedes modificarlo.

🌐index.html

La web del organigrama. Lee data.json al cargar y pinta las cards, filtros, búsqueda, etc. Si actualizas el JSON, la web refleja los cambios al instante.

💡
Importante: no hay base de datos en vivo
Cada actualización requiere regenerar el data.json y subirlo a GitHub. La Lista de SharePoint es solo la fuente de verdad para ti, no se conecta directamente con la web.

Procesar CSV y fotos

Dóndeconversor.html Tiempo1-2 min

Arrastra el CSV al conversor. Después aparecerá un segundo paso para arrastrar las fotos. Sigue el flujo:

1
Arrastra el CSV exportado a la zona "Arrastra el .csv o .xlsx aquí"
2
El conversor lo procesa y muestra "X personas procesadas". Si hay errores (columnas faltan), te lo dice en rojo.
3
Aparece automáticamente un Paso 3 (opcional): arrastra la carpeta de fotos. Si no tienes fotos nuevas, puedes saltarlo.
4
El conversor procesa las fotos (barra de progreso). Te muestra cuántas matcharon con personas y cuántas no.

Revisa el resumen: personas totales · con foto · sin foto. Si algo no cuadra, vuelve al paso 1.
🔒
Tus datos no salen de tu navegador
El conversor procesa todo localmente. Las fotos no se suben a ningún servidor — quedan en memoria del navegador hasta que descargas el JSON.
⚠️
Si una foto no matchea con nadie
El conversor te lista los archivos cuyo nombre no coincide con ningún ID M365. Renombra el archivo correctamente y vuelve a empezar desde el paso 5.

03 · TROUBLESHOOTING

Problemas comunes

Las dudas y errores más típicos. Click en cada uno para ver la solución.

No veo los cambios después de subir el data.json

Caché del navegador. GitHub Pages ya publicó los cambios pero tu navegador sigue mostrando la versión vieja.

Soluciones:

  • Ctrl + Shift + R (Windows) o Cmd + Shift + R (Mac): hard refresh
  • Abrir la URL en modo incógnito (Ctrl+Shift+N): descarta caché
  • Esperar 2-3 minutos: GitHub Pages a veces tarda en propagar
Una persona aparece sin foto aunque la subí al conversor

El nombre del archivo no matchea con su ID M365.

Revisa que la foto se llame exactamente como su ID M365 en la Lista. Por ejemplo, si su ID es Monco-Ferre Almudena, la foto debe llamarse Monco-Ferre Almudena.jpg (o similar — el conversor tolera espacios, guiones, mayúsculas).

Cuando proceses las fotos en el conversor, te muestra qué archivos NO matchearon con ninguna persona. Renombra esos y vuelve al paso 5.

El conversor dice "Falta columna requerida"

El conversor necesita estas columnas exactas en el CSV: ID M365, Nombre, Surname, Rol/Posición, Team.

Si alguna falta, vuelve a la Lista de SharePoint y comprueba que existen con ese nombre exacto. Si el nombre cambió, dímelo y actualizo el conversor para que lo reconozca.

El data.json pesa más de 10 MB

Probablemente las fotos originales son muy pesadas. El conversor las redimensiona a 400px pero JPEG calidad 85% conserva mucho detalle.

Opciones:

  • Comprime las fotos originales antes de subirlas (a ~300×300, < 100 KB cada una)
  • Usa un servicio como tinypng.com para comprimir en lote
  • Si pesa más de 100 MB, GitHub lo rechazará — habrá que usar otra estrategia
GitHub me dice que el commit es muy grande

GitHub permite archivos hasta 100 MB. Si tu data.json supera eso, las fotos están sin comprimir bien.

Reduce el tamaño máximo en el conversor (actualmente 400px). Pídeme bajarlo a 300px y se reducirá ~40% el peso.

No me deja exportar la Lista a CSV

Necesitas permisos de lectura sobre la Lista. Si no aparece el botón "Exportar":

  • Asegúrate de tener al menos permisos de Lectura sobre la Lista
  • Refresca la página (a veces SharePoint tarda en cargar el toolbar)
  • Alternativa: exporta a Excel (.xlsx) — vale igual
¿Cómo cambio entre modo claro y modo oscuro?

En el header del organigrama, arriba a la derecha, hay un botón con un icono de sol/luna ☀️🌙. Click para alternar.

Tu preferencia se recuerda en el navegador (localStorage). Si quieres resetearlo: abre la consola del navegador (F12), pestaña "Application" → "Local Storage" → borra la entrada organigrama-theme.

Quiero añadir/eliminar una columna en la Lista

Puedes añadir cualquier columna que quieras a la Lista — el conversor solo lee las que reconoce. Pero si quieres que esa columna aparezca en el organigrama, dímelo y actualizo el código del conversor + el index.html.

Personas que NO aparecen en el organigrama

El organigrama filtra automáticamente:

  • Estatus = Inactivo → no se ven, no se cuentan
  • Nombre contiene "Demanda" (ej: "Manager 1 Zz Demanda") → no se ven

Los demás siempre aparecen, independientemente de su estatus (Activo, Baja Temporal, etc.).
